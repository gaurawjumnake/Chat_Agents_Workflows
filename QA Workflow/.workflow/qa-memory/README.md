# QA Memory (Obsidian-Compatible)

Persistent QA knowledge lives here and should be referenced by path in chat workflows.

## Structure

- `bugs/`: reproducible defects and root causes
- `security/`: security findings and mitigations
- `healing/`: flaky recovery analyses and approvals
- `patterns/`: reusable QA validation patterns
- `regression/`: targeted regression history and rationale
- `specs/`: QA notes linked to specs
- `snippets/`: snippet usage notes and variants

## Rules

- Keep notes short and searchable.
- One note per event/decision.
- Link related spec, test run, and environment.
- Prefer references over retyping context.
