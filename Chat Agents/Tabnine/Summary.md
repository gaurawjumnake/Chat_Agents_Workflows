### Tabnine

**Type:** IDE Extension  
**Model:** Proprietary + optional BYOK (enterprise)  
**Pricing:** Dev $9/mo · Enterprise (custom, on-prem available)  
**Repo/Docs:** https://docs.tabnine.com

The privacy-first AI coding assistant. The only tool in this list with true air-gapped, on-premise deployment — code never leaves your infrastructure. In operation for years; mature, stable, and widely deployed in regulated industries (finance, healthcare, defence, government). The privacy trade-off: its completion quality is behind cloud-based frontier tools like Cursor or Claude Code. Private codebase models: fine-tunes on your organisation's code for domain-specific suggestions.

**When to use:**
- Data residency requirements — no code can leave your network
- Regulated industries where cloud AI tools are prohibited
- Enterprise-wide standardisation where compliance and admin controls matter more than raw capability
- You want a fine-tuned model trained on your own codebase

**When NOT to use:**
- You want best-in-class AI reasoning — cloud tools are significantly stronger
- You need agentic multi-file editing — Tabnine is completion-focused
- Small teams or individuals without compliance requirements — cloud tools are cheaper and better

**Accuracy:** Below frontier cloud tools. Completion quality is good but not state-of-the-art.

**Latency:** Very fast — sub-50ms completions. Best inline latency in this list.

**Token efficiency:** Medium. Completion-focused; limited context window on local models.

**IDE support:** VS Code, JetBrains, Vim/Neovim, Eclipse, Visual Studio, and more — broadest IDE support of any tool here.

**MCP:** No

**Privacy:** On-prem / air-gapped. Zero data transmission outside your network (Enterprise). Zero data retention by default.

**Limitations:**
- Completion quality below cloud frontier tools
- Agentic features are basic compared to Cursor, Cline, or Claude Code
- Enterprise pricing requires a sales call — no self-serve on-prem
- Local models require significant server hardware
